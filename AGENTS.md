# Repository Guidelines

## Project Structure
This repository is a single-module Android LSPosed/libxposed project. The main
module code lives in `app/src/main/java/com/example/smoothisland/`, with the
current entry point implemented in `XposedInit.kt`.

Important project files:

- `app/src/main/java/com/example/smoothisland/XposedInit.kt`: installs the
  SystemUI hooks and generates smooth capsule outlines.
- `app/src/main/resources/META-INF/xposed/java_init.list`: declares the
  libxposed entry class.
- `app/src/main/resources/META-INF/xposed/module.prop`: declares libxposed API
  101 metadata and static scope behavior.
- `app/src/main/resources/META-INF/xposed/scope.list`: scopes the module to
  `com.android.systemui` and `miui.systemui.plugin`.
- `app/src/main/AndroidManifest.xml`: Android application metadata used by the
  module APK.
- `.github/workflows/build.yml`: CI workflow for debug or signed release APKs.
- `analysis_report_miui_features.md`: research notes about MIUI/HyperOS dynamic
  island rendering and smooth-corner behavior.

## Build Commands
This repository does not include a Gradle wrapper. Use an installed Gradle that
is compatible with the Android Gradle Plugin declared in the root build file.
CI currently uses Gradle 9.1.0 and JDK 17.

- `gradle assembleDebug`: build a debug APK for local LSPosed testing.
- `gradle assembleRelease`: build a release APK; signing is used only when the
  required environment variables are present.
- `gradle clean`: remove Gradle build outputs.

The Android configuration currently targets SDK 34, requires minSdk 31, and
uses Java 17 compile options.

## Runtime Behavior
Host-side outline hooks should only run in the `com.android.systemui` process.
Plugin-side background/stroke hooks target `miui.systemui.plugin` classes. Keep
these paths separate; broad hooks against framework drawing APIs can otherwise
affect unrelated processes.

The current hook strategy is:

- Hook `android.graphics.Outline.setRoundRect(int, int, int, int, float)`.
- Detect capsule-like outlines where the radius is approximately half the view
  height.
- Replace those outlines with a `Path` generated by
  `androidx.graphics.shapes.RoundedPolygon.pill(...).toPath()`.
- Hook MIUI's dynamic-island outline provider class when available and normalize
  its returned capsule outline.
- Read `persist.smoothisland.smoothing` for the smoothing coefficient, clamped
  to `0.0..1.0`, with a default of `0.8`.

Generated paths are cached by width, height, and smoothing value. Keep the cache
small because this code runs inside SystemUI.

Plugin-side stroke handling:

- Hook `DynamicIslandBaseContentView.updateMedianLuma(float)` in
  `miui.systemui.plugin`.
- Let MIUI draw its original stroke while the island geometry is changing.
- After the background bounds remain stable for a short debounce window,
  replace the plugin `GradientDrawable` stroke with a smooth stroke drawable
  using the same capsule path generator as the host outline hook.

## Coding Guidelines
Use the style already present in `XposedInit.kt`: small private helpers, clear
guards, and minimal side effects inside hooks. Avoid broad class-name scanning
or logging in hot rendering paths unless it is needed for diagnosis.

When editing hook code:

- Fail closed when MIUI-specific classes or methods are absent.
- Keep reflection and hook installation inside guarded code paths.
- Avoid crashing SystemUI from optional compatibility logic.
- Preserve the module entry class name unless `java_init.list` and ProGuard
  rules are updated at the same time.

## Testing Guidelines
There is no automated test suite in this repository. For changes that affect
hook behavior, validate at least this path:

1. Run `gradle assembleDebug`.
2. Install the APK on a rooted MIUI/HyperOS device with LSPosed/libxposed
   support.
3. Enable the module and confirm the scope is `com.android.systemui`.
4. Restart SystemUI or reboot.
5. Verify the island outline visually and check LSPosed/Xposed logs for
   hook-loading errors.

If tests are added later, place JVM tests under `app/src/test/java/` and Android
instrumentation tests under `app/src/androidTest/java/`.

## Release Notes
The GitHub Actions workflow builds a signed release APK when signing secrets are
available. Without those secrets it builds and uploads a debug APK.

Release builds enable R8. Keep rules for the libxposed entry point and
`XposedModule` subclasses in `app/proguard-rules.pro`.
